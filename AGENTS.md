# AGENTS.md

## Project purpose
This repository publishes an anonymized, public-interest evidence report about an employment-related dispute. Its purpose is to support transparent verification, institutional review, and responsible media/public understanding without exposing sensitive personal data.

## Core rules
- Preserve anonymization in all edits and newly added content.
- Do not add sensitive personal data (PII), private identifiers, or raw confidential artifacts.
- Present all wrongdoing-related statements as claims/allegations unless supported by final, official legal determination.
- Use careful, formal, legally safe language; avoid inflammatory or absolute phrasing.
- Keep factual precision: prefer verifiable wording, clear attribution, and dated context.

## Content strategy
- Treat Markdown files as the source of truth for substantive content.
- Treat HTML pages as human-friendly entry points for readers, media, and institutions.
- Prefer additive and organizing changes over destructive edits.
- Do not remove existing documentation unless there is explicit, justified necessity.
- Every change should improve readability, credibility, and public usefulness.

## Structure
- Maintain parallel language structure across PL / NL / EN wherever applicable.
- When adding or updating key pages/sections, keep cross-language consistency in scope and intent.
- Ensure navigation remains clear between entry pages and source documents.
- Keep filenames, links, and references stable and case-accurate.

## Review priorities
1. Legal and privacy safety (anonymization, no sensitive data, allegation framing).
2. Consistency with Markdown source-of-truth documents.
3. Clarity and accessibility for public, media, and institutions.
4. Language parity across PL / NL / EN.
5. Link integrity and non-destructive change management.
